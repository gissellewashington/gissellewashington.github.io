Lab 1: The Artemis Board and Bluetooth

Part 1) Objective

The objective of the first part of the lab was to familiarize myself with the Arduino IDE software and the Artemis Nano microcontroller. I learned how to set up the Artemis and install the necessary libraries and packages to the Arduino IDE to run basic scripts on the Artemis.

Setup

The first step to setting up the Artemis board was installing the Arduino Core for Apollo 3. Using these instructions Artemis Development with Arduino , I was able to install the Sparkfun Apollo 3 package. The necessary source files for the Arduino core and Apollo 3 libraries were in the installation. Then I had to connect the Artemis Board to my computer using a USB-C cable.

Tasks

Blink It Up!

Through this task, I was able to load the blink example code by navigating to File->Examples->01.Basics->Blink on the Arduino IDE. Then I uploaded the code to make sure the module could receive the code and make sure the core was installed correctly. Here is a video of the Artemis Board responding to the Blink code:

Example04_Serial

Through this task, I could send a serial message to the Artemis via the serial monitor and have the message echoed back to me as the output. I could navigate to the code by following File->Examples->Artemis Examples, and run Example4_Serial. The TA advised me to set the baud rate to 115200 baud which increases the number of signal elements per second.

Example02_analogread

In the analog read task, I tested the temperature sensor with the Artemis Board by placing my finger on the board to see variations in the temperature by lifting my finger off the board. I accessed the example code to test the temperature by following File->Examples->Artemis Examples, and run Example2_analogRead. Additionally, I modified the serial print line to replace temp_raw with getTempDegF() in line 63. This allowed the temperature to be in units of Fahrenheit instead of raw ADC counts for a more physical intuition of the output.

Example1_MicrophoneOutput

In the microphone example, I was able to test the accuracy of the Pulse Density microphone by whistling into the Artemis Board and observing changes in frequency on the serial monitor. I accessed the example code by following File->Examples->PDM, run Example1_MicrophoneOutput. First I observed the frequency output without the whistle which was an order of magnitude of 500 then followed by whistling directly over the Artemis and received a higher frequency measurement at an order of magnitude around 1900.

Part 2) Introduction

The purpose of this lab was to provide an introduction to sending and receiving data to and from the Artemis Board via Bluetooth communication with a computer. In this lab, I learned how to write program commands in Arduino IDE, use Jupyter lab notebook to write and read serial messages between the Artemis Board and computer using the ArtemisBLEController operations, and collect data from the temperature sensor and Pulse Density Microphone.

Setup

To begin the lab, I had to install Python 3 onto my computer and create a project directory for my Arduino files. The installation was needed to support the lab 2 codebase used to establish the communication channel between the computer and Artemis Board through the BLE Controller operations. I discovered an issue with accessing the latest version of Python due to previously installed versions already on my computer only being recognized by the command prompt. The lab TAs helped resolve the issue by removing the previously installed versions and activating the virtual environment through the command prompt and not Powershell.

I used link to install the latest version of Python. Additionally, I installed pip directly by typing the following in the command line interface:

python3 -m pip install --user virtualenv

I navigated to my project directory and created a new virtual environment named “FastRobots_ble” by typing the following into the command prompt:

python3 -m venv FastRobots_ble

Next, I downloaded the lab 2 codebase directly to the project directory. For using the lab 2 scripts, I needed to activate the virtual environment and start the Jupyter Notebook server by typing the following into the command prompt:

.\FastRobots_ble\Scripts\activate

jupyter lab

Lastly, I installed the ArduinoBLE from the library manager into the Arduino IDE. The Python and Artemis packages provided the basic code to establish the connection between the Artemis board and the computer. I uploaded the ble_arduino.ino sketch into the Artemis Board which caused it to print its MAC address.

artemis_address: 'c0:83:75:6c:b8:3c'

Configuration

In the configuration file connections.yaml, I replaced the Artemis address with the printed MAC address from the serial monitor after uploading the ble_arduino sketch. Following this step, I generated a Universally unique identifier (UUID) which helps differentiate between different kinds of data sent between the Artemis and the computer. In the ble_arduino sketch, I replaced the BLEService UUID line with the generated UUID and replaced the ble_service line with the generated UUID in the configuration file. This step was essential to making sure that the connection channel was made between my computer and my Artemis Board and not another classmate’s Artemis Board. The class ArtemisBLEController was ready to use the identifiers in the connections.yaml file to send and receive data and process commands to the Artemis Board.

Tasks

ECHO Command

In the Jupyter Notebook, I sent an ECHO command to the Artemis Board with a string value using the class ArtemisBLEController. The Artemis Board sent an augmented string back to the computer after clearing the estring and writing a new estring message to the characteristic array shown in the code below.

GET_TIME_MILLIS Command

Similar to the echo command, I constructed a command in the Arduino ble sketch that would allow the Artemis to overwrite a string to the string characteristic and send the string data to the computer (see code below). The get_time_millis command would build upon the echo command by utilizing the built-in millis() function to get a timestamp of the form “T:12345”.

Notification handler

The notification handler is a callback function that receives a string value from the Artemis Board. The function receives a list of the data of the data as it keeps updating. The start notify function takes the uuid and the byte_array as inputs and returns an array of the output (see code below).

Method 1 - Task 4

For this task, I created a command in the ble sketch that conducts a loop that collects time stamp data to be sent to my computer. The values were collected for a few minutes and ranged in magnitude from the order of 10,000. The effective data rate transfer from this method collected was around 65 milliseconds.

int time_stamp[30]; // Time Stamp Array,
int temp_stamp[30]; // Temperature Array

SEND_TIME_DATA

In this task, I initialized a time and temperature array to store the data. In a new command named STORE_TIME_TEMP_READINGS, I looped through the indices of the array and updated each element with a corresponding time stamp value and temperature data value. Then as the data was collected, I created a SEND_TIME_DATA command which looped through each array element and sent the data over to my computer to be processed.

Method 2 - GET_TEMP_TIME_READINGS

In this task, I created a command titled GET_TEMP_TIME_READINGS which would loop through both the time stamp and temperature array concurrently and send each temperature reading with its corresponding time stamp as a concatenated string to the computer. The notification handler would collect the incoming data and separate the strings into two separate lists by splitting along the “|”. This task contrasts the previous method in the get_time_millis command because all the data is collected and stored first locally to the Artemis and then sent to the computer collectively instead of individually after each data collection.

Differences between the two methods

The main difference between the two methods is the magnitude of the effective data transfer rate. I noticed in the second method that storing the data first in the individual arrays and then sending it concurrently had a transfer rate of around 1 milli seconds compared to the first method which was around 1 ms. The time complexity of the second method is much smaller than the time complexity of the first method meaning that larger sets of sensor data can be sent can be sent in shorter time frames than the first method. There may be a disadvantage to the second method given the space complexity of the Artemis board and storing all the sensor data. The Artemis board has 384,000 bytes of RAM and if it was used in its entirety it would account for 96000 float values or 48000 double values.

Discussion

In this lab, I learned how to use the Jupyter Notebook to receive data from the Artemis board and send commands to the Artemis Board, I became familiar with programming the different commands in the Arduino sketch, and I learned how to connect the Artemis Board to the computer using Bluetooth communication. It will be interesting to see how Bluetooth communication is used in the following labs to communicate data to the computer and be processed to make decisions.
